# Auto Update GoldHEN (PS4 Homebrew)

هذا المشروع هو تطبيق PS4 Homebrew متكامل مصمم لتحديث ملفات GoldHEN تلقائياً بمجرد تشغيله.

## الوظائف
- نسخ `payload.bin` إلى `/data/payload.bin`
- نسخ `goldhen.bin` إلى `/data/GoldHEN/payloads/goldhen.bin`
- إنشاء المجلدات الناقصة تلقائياً.
- استبدال الملفات الموجودة إجبارياً.
- إظهار إشعارات النظام عند النجاح أو الفشل.
- تسجيل العمليات في ملف `log`.

## هيكل المشروع
- `source/`: يحتوي على الكود البرمجي C++.
- `include/`: يحتوي على ملفات الترويسة والتعريفات.
- `assets/`: ضع ملفات الـ BIN المراد نسخها هنا قبل البناء.
- `sce_sys/`: يحتوي على موارد الـ PKG (أيقونات، معلومات).
- `.github/workflows/`: يحتوي على سير عمل GitHub Actions للبناء التلقائي.

## كيفية البناء

### البناء المحلي (Local Build)
يتطلب المشروع وجود **Orbis Toolchain** (Open Orbis SDK).
1. تأكد من ضبط متغير البيئة `ORBISDEV` ليشير إلى مسار الـ SDK الخاص بك.
2. ضع ملفات `payload.bin` و `goldhen.bin` الأصلية في مجلد `assets/`.
3. قم بالانتقال إلى مجلد المشروع في الطرفية.
4. قم بتشغيل الأمر:
   ```bash
   make
   ```
5. سينتج ملف `AUG000001.pkg` جاهز للتثبيت على جهاز PS4 المهكر.

### البناء باستخدام GitHub Actions

لقد تم تضمين سير عمل GitHub Actions في ملف `.github/workflows/build.yml` يقوم ببناء المشروع تلقائياً عند الدفع (push) إلى فرع `main` أو عند إنشاء طلب سحب (pull request) أو عند التشغيل اليدوي.

**ملاحظات هامة لـ GitHub Actions:**
- يجب عليك توفير طريقة لتثبيت `Orbis Toolchain` في بيئة GitHub Actions. الخطوة الحالية في `build.yml` هي مجرد **عنصر نائب (Placeholder)**. ستحتاج إلى استبدالها بخطوات فعلية لتنزيل وتثبيت الـ Toolchain الخاصة بك أو استخدام صورة Docker مخصصة تحتوي عليها.
- بعد اكتمال البناء بنجاح، سيتم رفع ملف الـ PKG كـ `artifact` يمكنك تنزيله من صفحة GitHub Actions الخاصة بمستودعك.
